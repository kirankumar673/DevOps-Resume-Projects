# Resume Points — Project 01: Static Website on AWS S3

---

## Fresher

- Hosted a static website using Amazon S3 Static Website Hosting with zero server management.
- Configured S3 bucket policies using IAM JSON policy to grant public read access to website objects.
- Disabled S3 Block Public Access settings and applied least-privilege bucket policies for controlled public access.
- Deployed production-ready HTML5 files with proper `lang`, `charset`, and `viewport` meta tags to AWS S3.

---

## Experienced DevOps Engineer

- Designed and deployed a serverless static website hosting solution using Amazon S3, eliminating server management overhead.
- Configured S3 bucket policies, static website hosting endpoints, and public access controls following AWS security best practices.
- Identified HTTP-only limitation of S3 static hosting and documented a CloudFront + ACM upgrade path for HTTPS enforcement.
- Managed cloud infrastructure cleanup procedures to prevent unnecessary AWS billing.

## How to Explain in an Interview (30 Seconds)

"I built a static website hosting solution on AWS S3. I created an S3 bucket, disabled public access blocks, uploaded HTML and CSS files, enabled Static Website Hosting, and applied a bucket policy for public read access. I also identified that S3's native endpoint is HTTP-only and documented the upgrade path using CloudFront with an ACM certificate for HTTPS — which is what you'd do in a real production setup."
